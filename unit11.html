Lecture Notes

Mapping Concepts

The map school assigned reading provides a great overview of basic mapping concepts. The main issue we are going to concern ourselves with is vector data. In the video, I discuss the following image of storing the geodata for the boundaries (lat, lon) for the state of Colorado.

Two important things to understand about how Tableau handles boundaries are: * coordinate data must be in decimal degree format * the polygon data must also have a field that represents which polygon the data point belongs to and the draw order for the point. Both of these fields should be dimensions.

In the videos, we take two types of data from the Maine Office of GIS website. We get geocoded Maine town boundary data in two different formats. One file is a shapefile, which was developed by Esri and used in ArcView GIS. The other file is a KML file which is commonly used by Google Earth.

We should probably discuss why we might want to work with these files instead of using the native mapping capabilities in Tableau. Tableau, through its map data partners is improving their geodata fairly regularly but they don't have a lot of data on smaller population towns. For example, in Cumberland County, Tableau only knows about Portland, South Portland, Brunswick, Westbrook, and Gorham.

There are a few options I have for external mapping in Tableau. First, if I don't want to use the default maps that come with Tableau, I can look for different WMS servers. I can also purchase additional maps and features from Urban Mapping.

Luckily, most US states provide tons of geodata for free on the internet. In the videos, I walk through a quick process of converting KML boundary data into push-pin points using a third-party conversion tool. You don't need to actually do this if you do not wish as it involves installing additional software.

I also walkthrough converting a shapefile while keeping the boundary data intact. The shapefiles provided by Maine GIS don't use geographic coordinates. They use the North American Datum(NAD) which uses measurements from an "anchor point." Specifically, it uses the NAD83 / UTM zone 19N as the coordinate system and projection. For example, the geographic coordinate -69, 62 is stored as 500000, 6874180.147719. Probably the four most popular ways for identifying coordinates on a map are: WGS84(which uses lat and long); Universal Trans Mercator; the British National Grid; and State Plane Systems

Projection, in this case, means that we are taking 3D coordinates and plotting them on a 2D surface. This causes a variety of issues -- for example, we often use Mercator projections for maps used for navigation. Mercator projections also cause distortion, as illustrated by this interactive mercator puzzle. NAD83 / UTM zone 19N uses a Mercator coordinate system.

To simplify the issue, our challenge is simply to convert the coordinates to the geographic coordinates understood by Tableau. There are five videos in this unit. In the second video, I use Google Earth to unarchive a .KMZ file into a .KML file. There are many KML converters out there. The one I use in the video is Zonum's kml2x web based tool. I don't normally use a tool like this but wanted to illustrate how to quickly do the conversion with minimal software installation. The downside of the tool is that boundary polygons are converted to points resulting in significant data loss.

In the fourth video, I use the shapefile from Maine GIS but this time I explicitly want to keep the boundary data. We are used to seeing bounded polygon maps for things like flu outbreaks, voting results, crime statistics, etc. but with the increasing prevalence of location based data due to social/local apps, even business mapping has been moving away from simple push-pin type maps to bounded polygons. Tableau does have good U.S. boundaries for zip codes but not for towns. In Maine, a majority of the town boundaries are similar/identical to their zip boundaries but this isn't always the case. To perform the conversion, I follow the steps below:

  • use QGIS an open-source GIS application to convert the coordinate system from UTM to WGS (specifically from NAD83 / UTM zone 19N to EPGS:4326).
  • use a Tableau community provided R program to convert the previously converted shapefile into a csv format readable by Tableau.
  • use Excel to filter out the data that doesn't apply to Cumberland county.

In the fifth and final video, I quickly map the cleaned data and use a wms map from the available Maine GIS wms maps.

If you are interested in more GIS knowledge, there is a Coursera MOOC - Maps and the Geospatial Revolution. The course uses ArcGIS, which is commercial software. If you are more interested in a shorter QGIS focused course, the Newbies Quantum GIS Tutorial is a good starting point.
